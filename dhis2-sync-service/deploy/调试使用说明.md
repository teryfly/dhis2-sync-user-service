
聚合逻辑：
- 用 `LinkedHashMap<user_id, UserWithRolesDTO>` 合并多行
- 每行把 `role_id` 追加到 `roleCodes`

日志（INFO）：
- `从数据库查询到 {} 条用户记录（含角色关联）`

排错建议：
- 如果用户表有数据但返回 0：检查 `user_rolejc_mj` 是否存在关联（INNER JOIN 会过滤无角色用户）
- 某用户角色列表异常：检查 join 结果是否重复/role_id 是否为空

## 8. DHIS2 调试（Client 层）

所有 Client 使用 `RestClientConfig` 创建的 `RestClient`：
- baseUrl：`dhis2.base-url`
- 默认头：
  - `Authorization: Basic base64(username:password)`
  - `Content-Type: application/json`

### 8.1 角色查询与创建

查询：
- GET `/api/42/userRoles?filter=code:eq:{code}&fields=id`
- 解析：`userRoles[0].id`

创建：
- POST `/api/42/userRoles`
- body：`{name, description, code}`

Client DEBUG 日志：
- `DHIS2 角色查询: code={}, found={}`
- `DHIS2 角色创建请求: code={}, response={}`

排错建议：
- 401/403：账号密码错误或权限不足
- 404：baseUrl/路径错误或 DHIS2 API 版本不支持（硬编码 `/api/42`）
- JSON 解析失败：DHIS2 返回非 JSON（可能是 HTML 错误页），检查响应体

### 8.2 用户查询与创建

查询：
- GET `/api/42/users?filter=username:eq:{username}&fields=id`
- 解析：`users[0].id`

创建：
- POST `/api/42/users`
- body 包含：
  - username/password/firstName/surname
  - userRoles: `[{"id": roleUid}, ...]`
  - organisationUnits: `[{"id": orgUid}, ...]`

Client DEBUG 日志：
- `DHIS2 用户查询: username={}, found={}`
- `DHIS2 用户创建请求: username={}, response={}`

常见失败：
- DHIS2 校验失败：缺少角色或组织机构、字段不合法等（会以 4xx 返回并抛异常）
- username 不允许重复：会被 `findUserByUsername` 规避，但如果并发或历史脏数据可能仍失败

### 8.3 机构查询

查询：
- GET `/api/42/organisationUnits?filter=code:eq:{code}&fields=id`
- 解析：`organisationUnits[0].id`

Client DEBUG 日志：
- `DHIS2 机构查询: code={}, found={}`

## 9. 业务同步调试（Service 层）

### 9.1 RoleSyncService

流程：
1) 从 DB 全量读取 roles
2) 每条 role：
   - 先 `findRoleByCode(code)`
   - 存在 → SKIPPED
   - 不存在 → `createRole(...)` → SUCCESS
   - 异常 → FAILED（不影响后续循环）

日志（INFO/ERROR）：
- `开始同步角色...`
- `角色已存在,跳过: code={}, name={}`
- `角色创建成功: code={}, name={}`
- `角色同步失败: code={}, name={}, error={}`（带堆栈）
- `角色同步完成: total={}, success={}, skipped={}, failed={}`

### 9.2 UserSyncService

流程：
1) 从 DB 全量读取 users（含 roleCodes 聚合）
2) 每个 user：
   - `findUserByUsername(username)` 存在 → SKIPPED
   - 不存在 → 解析依赖并创建：
     - roleCodes → 逐个 `findRoleByCode(roleCode)` 解析 roleUid
       - 找不到 → WARN 并跳过该角色
     - orgcode → `findOrgUnitByCode(orgcode)` 解析 orgUid
       - 找不到 → WARN，orgUnitUids 为空
     - 调 `createUser(...)`
   - 异常 → FAILED（不影响后续循环）

日志（INFO/WARN/ERROR）：
- `开始同步用户...`
- `用户已存在,跳过: username={}`
- `角色未找到,跳过该角色: roleCode={}, username={}`
- `机构未找到,创建用户时机构列表为空: orgcode={}, username={}`
- `用户创建成功: username={}, roleCount={}, orgUnitAssigned={}`
- `用户同步失败: username={}, error={}`（带堆栈）
- `用户同步完成: total={}, success={}, skipped={}, failed={}`

## 11. 常见问题与排查清单（按现象）

### 11.1 启动即报数据库连接错误

现象：
- HikariPool 初始化失败、Connect refused、timeout

排查：
1) DB 是否可达（nc/telnet）
2) `application.yml` 的 url/username/password 是否正确
3) 防火墙/路由是否放通
4) DB 是否允许该用户远程连接

处理建议：
- 如果仅需验证编译打包，可先不运行；
- 需要本地调试必须确保 DB 可连。

### 11.2 DHIS2 请求 401/403

排查：
1) `dhis2.username` / `dhis2.password` 是否正确
2) DHIS2 用户是否有创建 role/user 的权限
3) baseUrl 是否正确（含端口）

### 11.3 DHIS2 请求 404

排查：
1) baseUrl 是否正确
2) DHIS2 是否支持 `/api/42`（版本不匹配会导致路径不存在）

### 11.4 用户创建失败但角色同步正常

可能原因：
- orgUnit 为空且 DHIS2 强制要求 organisationUnits
- roleUids 为空且 DHIS2 强制要求 userRoles
- username 规则/重复冲突
- DHIS2 侧字段校验（查看异常堆栈中的响应体）

建议：
- 先确认 `orgcode` 在 DHIS2 organisationUnits 中能通过 code 查到
- 确认角色 code 已成功同步到 DHIS2

### 11.5 定时任务不触发

排查：
1) `Dhis2SyncApplication` 是否有 `@EnableScheduling`
2) `sync.cron` 是否存在且语法正确
3) 应用是否持续运行（不是运行后立刻退出）
4) 时区问题（若部署环境时区非预期）

## 12. 调试建议（不改代码的前提）

1) 调整 cron 为每分钟触发，观察日志链路是否完整。
2) 将日志级别调整为 DEBUG（可选）：
   - 当前 logback 配置默认 root=INFO；
   - 若要更细粒度观察 DHIS2 Client DEBUG 日志，可在 `logback-spring.xml` 增加：
     - `com.dhis2sync` 设为 `DEBUG`
3) 观察 `./logs/sync.log` 与滚动文件，确认生产路径下日志落盘正常。

## 13. 推荐的开发调试流程（一步步执行）

1) 编译：
   ```bash
   mvn clean compile
   ```
2)（可选）改 cron 为每分钟：
   - `sync.cron: "0 * * * * ?"`
3) 运行：
   ```bash
   mvn spring-boot:run
   ```
4) 观察日志顺序：
   - Scheduler 触发
   - Orchestrator 开始
   - RoleSyncService 开始/逐条处理/结束
   - UserSyncService 开始/逐条处理/结束
   - Orchestrator 汇总与耗时
5) 若异常：
   - 优先看 ERROR 堆栈（Repository 连接、Client HTTP 错误）
   - 根据上文排查清单定位网络/权限/数据问题
6) 调试完成后：
   - cron 改回 `"0 0 2 * * ?"`
   - 打包：
     ```bash
     mvn clean package -DskipTests
     ```
